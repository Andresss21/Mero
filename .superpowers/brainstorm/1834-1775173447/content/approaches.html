How do we structure the project?

All three use Vite. The difference is how granular the module split goes.

A

Flat partials — HTML split, CSS/JS shared

Each section is an HTML file. One global CSS file, one global JS file. Vite assembles everything.

src/
  pages/
    index.html
    products.html
  sections/
    hero.html
    nav.html
    productos.html
    eventos.html
    ...
  styles/
    main.css          ← everything here
  scripts/
    main.js           ← everything here

Pros

  • Closest to what you have now
  • Easiest to get running
  • One CSS file = easy global changes

Cons

  • CSS/JS still grows into one big file
  • Harder to migrate section-by-section to Next.js
B

Full modules — HTML + CSS + JS per section

Each section lives in its own folder. Shared globals for tokens and layout chrome. Each folder ≈ one future Next.js component.

src/
  pages/
    index.html
    products.html
  sections/
    hero/
      hero.html
      hero.css
      hero.js
    nav/
      nav.html
      nav.css
      nav.js
    productos/
      productos.html
      productos.css
      productos.js
    eventos/
      ...
  shared/
    tokens.css        ← variables, reset
    animations.js     ← GSAP, Lenis setup
    cursor.js

Pros

  • Each section is fully self-contained
  • 1:1 mapping to future Next.js components
  • Easy to work on one section without touching others

Cons

  • More files upfront
  • Need to be careful with shared state (scroll, cursor)
C

Hybrid — modules for features, shared for layout chrome

Nav, footer, cursor, preloader stay in shared/. Feature sections (hero, products, events…) each get their own folder. CSS split into tokens + per-section.

src/
  pages/
    index.html
    products.html
  layout/
    nav/
    footer/
    cursor/
    preloader/
  sections/
    hero/
    productos/
    eventos/
    galeria/
    arma-rig/
    ...
  shared/
    tokens.css
    animations.js

Pros

  • Layout chrome separated from page content
  • Cleanest mental model
  • Best Next.js migration path (layout → app/layout.tsx)

Cons

  • Slightly more upfront structure to set up

My recommendation: C — nav/footer/cursor as layout, feature sections as modules. Maps cleanly to Next.js app router later.
